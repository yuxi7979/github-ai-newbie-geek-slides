MENTAL MODEL

先别怕代码：GitHub 是“项目时间机器 + 公开展柜”。

Repository 是项目文件夹；commit 是一次存档；branch 是平行宇宙；pull request 是“请大家看看我改得怎么样”。

repo = 项目 commit = 存档 PR = 变更申请
> git init
> git add .
> git commit -m "first story"
> git push origin main
✅ your idea is online
MAP OF GITHUB

新手先掌握这 8 个入口，就够用了。

Profile

你的技术名片：头像、简介、置顶项目。

Repository

项目主页：代码、README、历史记录。

README

项目说明书：给人类看的第一屏。

Issues

需求、bug、待办事项的公开看板。

Pull Requests

提交修改、接受 review、合并成果。

Actions

自动测试、自动部署、自动检查。

Stars / Forks

收藏项目，或复制一份自己改。

Pages

把静态网页直接发布成网站。

LOCAL RUNBOOK

把别人 repo 在本地跑起来：像拆盲盒，但有套路。

STEP 1

Clone

git clone <url>
cd <repo>

STEP 2

Read

先看 README：安装、运行、环境变量、示例截图。

STEP 3

Install

npm install
pip install -r requirements.txt

STEP 4

Configure

复制 .env.example，填入必要配置，别把密钥 push 上去。

STEP 5

Run

npm run dev
python app.py

STEP 6

Debug with AI

把报错、系统环境、已尝试步骤交给 Copilot 解释。

COMMAND FLOW

最常用 push 流程：保存、检查、提交、上传。

git status看看哪些文件变了
git add .把修改放进待提交区
git commit -m "update README"写一条人能看懂的记录
git push origin main上传到 GitHub
AI PAIRING

VS Code + GitHub Copilot Free：新手的“项目翻译官”。

安装

VS Code 登录 GitHub，安装 GitHub Copilot / Copilot Chat 扩展，选择 Free 个人版。

提问

选中报错或文件，问：“请用非程序员语言解释这个项目怎么运行。”

执行

让 Copilot 给你命令，但每一步都先看懂：它要装什么、改什么、是否涉及密钥。

推荐 prompt

我是纯文科新手，请一步一步带我在本地运行这个 repo。先读 README 和 package 文件，告诉我需要安装什么、运行什么命令、如果报错如何定位。

README THAT SELLS

求职加分的项目，不是代码最多，而是别人最快看懂。

README 要回答：这是什么、解决什么问题、怎么运行、有什么截图、你学到了什么、下一步想做什么。

必须有

一句话简介、功能列表、技术栈、运行命令、截图或 demo。

最好有

项目背景、设计思路、AI 如何参与、遇到的坑和解决方式。

不要有

空 README、密钥、无法复现的步骤、只有“test project”的说明。
